# TASK SUMMARY: @schloss/gate & @schloss/core Web Crypto Runtime Decryption Engine

- FILE LOCATIONS: Central configurations in modules/core/src/ciphers.ts, Client-side stream decryptor in modules/gate/src/decrypt.ts, Server-side ingestion stream in modules/kern/src/ingest.ts
- [MAIN OBJECT] TABLE/SCHEMA: Binary Envelope Header Format comprising routing type byte (0: public, 1: individual, 2: group), key version (2-byte big-endian), IV envelope (12-byte initialization vector), IV data (12-byte base data initialization vector), enveloped DEK length (2-byte big-endian), and variable-length wrapped Data Encryption Key (DEK) bytes
- [SUPPORTING DATA] STRUCTURES: CipherEnvelopeHeader interface containing routingType, keyVersion, ivEnvelope, ivData, envelopedDekLength, and envelopedDek properties; DecryptionContext interface with optional privateKey and groupMasterKey properties; CIPHER_CONFIG object defining AES-GCM, ECDH, and AES-KW parameters alongside plaintext block sizes and encrypted block size math properties
- SYSTEM LOGIC/SIGNATURES: deriveBlockIv(baseIv: Uint8Array, blockIndex: number) returns incremented IV bytes; parseEnvelopeHeader(buffer: Uint8Array) returns parsed header fields or null; decryptAssetStream(encryptedStream, context) processes cipher blocks on-the-fly via transform stream logic; decryptDataEncryptionKey(header, context) extracts raw symmetric payload keys; createEncryptionStream(importDek, ivData, headerBytes) prepends header and partitions input into identical blocks during ingestion
- CODE/DEPS POLICY: Strictly zero trailing semicolons across all source files, single-quote string constants, unfolded long signatures, no external runtimes or pure JS cryptographic libraries utilizing native subtle crypto engine exclusively, streaming inputs mapped to 64KB block frames to guarantee hardware acceleration on mobile devices, errors during decryption instantly terminate the execution pipeline to prevent incomplete memory writes
